# EasyReader 书柜 UI 设计

## 背景

EasyReader 当前书柜是基础列表，已经支持导入 EPUB、打开书籍、长按选择和桌面快捷方式。用户希望书柜排布参考 Perfect Viewer：重点不是木纹背景，而是书籍和系列的网格排布方式。当前阶段优先优化两件事：

1. 书柜首页采用接近 Perfect Viewer 的系列堆叠网格。
2. 导入和展示本地 EPUB 元数据，尤其是封面、标题、作者和系列信息。

在线元数据刮削参考 Readest 的思路，但不放入本轮实现范围。Readest 的做法是本地导入阶段优先从 EPUB 内部读取 OPF 和封面，OpenLibrary / Google Books 等在线 provider 作为手动刷新元数据的增强能力。

## 目标

- 首页使用 3 列书柜网格，系列以堆叠封面显示，未归组单本以普通封面显示。
- 点进系列后显示系列内单本书籍网格。
- 每本书和每个系列都展示绿色阅读进度条。
- 书柜进度复用阅读页已经建立的总章节滚动桥结果，不重新设计进度算法。
- 自动归组和手动整理并存，手动整理优先级最高。
- 导入时尽量提取 EPUB 内部封面和基础元数据，让书柜第一屏可用。

## 非目标

- 不做在线元数据刮削、OpenLibrary / Google Books 集成或 API key 设置。
- 不做复杂书籍详情编辑页。
- 不做云同步、账号、OPDS、全文搜索、标签系统。
- 不重做阅读页进度算法，只消费已有持久化进度。

## 书柜信息结构

书柜展示层引入两个概念：

- `BookshelfEntry`：书柜首页的一格，可以是系列，也可以是单本。
- `BookshelfSeries`：一组书籍，包含系列名、书籍列表、主封面、堆叠封面、平均进度。

自动归组优先级：

1. 手动系列设置。
2. EPUB / Calibre series 元数据。
3. 用户自定义标题规则。
4. 内置默认标题规则，例如同一标题前缀加 `Vol.01`、`第 1 卷`、`01` 等。
5. 没有命中规则时作为单本显示。

手动整理入口保留在选择模式里：长按书籍进入选择后，提供“加入系列 / 新建系列 / 移出系列”。这些操作写入本地数据，优先级高于自动识别。拖拽调整顺序不放入本轮，系列内排序先使用手动序号、元数据卷号和标题自然排序。

## 自动归组规则接口

标题规则识别使用独立的 `SeriesGroupingRule` 接口，供内置规则和用户自定义规则共用：

- `id`：规则 ID。
- `name`：规则展示名。
- `pattern`：正则表达式。
- `enabled`：是否启用。
- `priority`：规则优先级，数值越小越先匹配。
- `builtIn`：是否为内置规则，内置规则可禁用但不可删除。

正则约定：

- 必须提供命名捕获组 `series`，作为系列名。
- 可选提供命名捕获组 `index`，作为系列内卷号或顺序。
- 命名捕获组采用 Java / Kotlin 正则语法，例如 `(?<series>.+?)\s+Vol\.?\s*(?<index>\d+)`。
- 没有匹配时继续尝试下一条规则。
- 同一个规则识别出的候选系列至少包含 2 本书时才折叠为系列，单本候选仍按单本显示，减少误归组。

内置默认规则：

- 英文卷号：匹配 `Book Title Vol.01`、`Book Title Volume 2`。
- 中文卷号：匹配 `书名 第 1 卷`、`书名 第01册`。
- 括号卷号：匹配 `书名 [01]`、`书名 (02)`。
- 保守数字后缀：匹配 `书名 01`，但只在同一候选系列出现 2 本及以上时生效。

用户自定义规则：

- 入口放在书柜更多菜单里的“系列归组规则”。
- 用户可以新增、编辑、启用、禁用和调整自定义规则顺序。
- 规则编辑页提供当前书柜标题的预览结果，保存前校验正则是否可编译、是否包含 `series` 捕获组。
- 无效正则不能保存；运行时如果历史规则因兼容问题变为无效，该规则会被跳过并在规则页标记错误。
- 自定义规则存入本地配置，优先级高于内置默认规则，但低于 EPUB / Calibre 元数据和手动整理。

## 首页布局

首页使用 `LazyVerticalGrid`，竖屏默认 3 列。网格项尺寸稳定，避免封面、标题、进度条加载后引起布局跳动。

系列项视觉：

- 主封面在最前，后面错位叠放 2 到 4 张后置封面。
- 右下或封面下方显示 `N 本` 数量角标，样式参考 Perfect Viewer 的黑底白字。
- 封面下方显示绿色系列进度条。
- 标题最多两行，超出省略。

单本项视觉：

- 显示封面或文字占位封面。
- 封面下方显示绿色单本进度条。
- 标题最多两行，超出省略。

背景本轮沿用当前 Material3 背景，不实现木纹背景；书柜效果主要靠网格、封面和堆叠排布完成。

## 系列内页布局

点击系列项进入系列内页：

- 顶部显示系列名并提供返回。
- 内容使用同样的 3 列网格。
- 每本书显示封面、标题、绿色单本进度条。
- 排序优先使用系列卷号；没有卷号时按标题数字自然排序，再退回标题排序。

系列内页中的已读规则：

- 未打开：0%。
- 阅读中：使用保存的总进度。
- `totalProgression >= 0.99`：视为已读完，按 100% 展示。

## 进度数据

书柜不重新计算 EPUB 章节权重。阅读页已经通过章节权重 + WebView 滚动桥得到总进度，并保存到 `ReadingProgress.totalProgression`。书柜只消费这个持久化字段：

- 单本进度 = `ReadingProgress.totalProgression`，为空时 0。
- 单本显示进度 = 小于 0 时归 0，大于 1 时归 1，达到 0.99 时归 1。
- 系列进度 = 系列内所有单本显示进度的平均值。

示例：10 本书中 1 本 100%，1 本 50%，其余 0%，系列进度显示 15%。

这保证书柜进度和阅读页底部总进度语义一致。

## 元数据与封面

导入阶段增强 `EpubImportService`：

- 继续读取 `META-INF/container.xml` 定位 OPF。
- 从 OPF 读取标题、作者、章节。
- 增加 series 元数据读取：
  - 优先 EPUB3 `belongs-to-collection` / `collection-type=series`。
  - 兼容 Calibre `calibre:series` 和 `calibre:series_index`。
- 增加封面解析：
  - 优先 manifest `properties="cover-image"`。
  - 再使用 OPF2 `<meta name="cover" content="...">` 指向的 manifest item。
  - 最后用图片 manifest 中包含 `cover` 的文件名做兜底。
- 封面保存到每本书私有目录，例如 `books/<bookId>/cover.<ext>`，并写入 `Book.coverPath`。
- 读取失败时不阻断导入，书柜显示文字占位封面。

封面解码和缩略：

- 本轮对 JPEG / PNG / WebP 等 Android 可解码位图做缩略，最长边限制为 512px，避免书柜大量图片占用内存。
- 原图小于限制时保留原始尺寸和格式。
- SVG 封面首版显示文字占位封面，不做 SVG 转位图。

## 组件边界

建议在 `ui/bookshelf` 内拆分少量组件，保持简单：

- `BookshelfScreen`：连接 ViewModel，保留导入、选择、打开书籍入口。
- `BookshelfGrid`：渲染首页条目网格。
- `SeriesStackItem`：渲染系列堆叠封面、数量和系列进度。
- `BookGridItem`：渲染单本封面、标题和单本进度。
- `SeriesBooksScreen` 或 `BookshelfContent` 内部状态：展示当前打开的系列。

ViewModel 输出适合 UI 的状态：

- `entries: List<BookshelfEntry>`。
- `selectedBookIds`。
- `openedSeriesId` 或由 UI 保存当前系列导航状态。
- `isImporting`、`message` 保持当前行为。

手动系列需要落库，数据层新增最小结构：

- 书籍的手动系列名或系列 ID。
- 系列内手动顺序。

自定义归组规则存入本地配置：

- `SeriesGroupingRule` 列表。
- 用户规则顺序和启用状态。
- 内置规则禁用状态。

本轮实现“加入系列 / 新建系列 / 移出系列”的基础落库；手动拖拽排序留到后续版本。

## 错误处理

- 元数据解析失败：使用文件名作为标题，作者为空，书籍仍可导入。
- 封面解析失败：`coverPath = null`，显示占位封面。
- 进度缺失：显示 0%。
- 系列识别冲突：手动设置优先；没有手动设置时选择更可信的 EPUB/Calibre 元数据。
- 自定义规则无效：保存前阻止；运行期跳过无效规则并保留书柜可用。
- 导入期间：沿用当前居中进度指示器和 Snackbar 消息。

## 测试计划

单元测试：

- 标题规则归组：卷号、`Vol.01`、中文“第 1 卷”等。
- 归组优先级：手动系列 > EPUB/Calibre series > 用户自定义标题规则 > 内置默认标题规则 > 单本。
- 自定义正则优先于内置标题规则。
- 无 `series` 捕获组的规则不能保存。
- 无效正则不会导致书柜崩溃。
- 单本候选不会被误折叠成系列。
- 系列进度平均值。
- `totalProgression >= 0.99` 视为 100%。
- 无进度为 0%。

导入测试：

- OPF `properties="cover-image"` 能写入 `coverPath`。
- OPF2 `meta name="cover"` 能写入 `coverPath`。
- 无封面时导入成功且 `coverPath = null`。
- Calibre series 元数据能被解析。

UI 回归：

- 书柜首页显示 3 列网格。
- 系列项显示堆叠封面、数量和绿色进度条。
- 点击系列进入系列内页。
- 长按仍进入选择模式，现有快捷方式入口不丢失。
- 导入后封面能在书柜中显示。

## 实施顺序草稿

1. 补数据读取：让 Repository / ViewModel 能拿到书籍进度，并生成单本进度。
2. 补元数据导入：提取封面和 series，失败时保留现有导入兜底。
3. 建立归组规则引擎：内置规则、自定义规则、规则校验和预览。
4. 建立归组逻辑：生成 `BookshelfEntry` 和系列平均进度。
5. 替换书柜列表为 3 列网格：先完成单本和系列项静态展示。
6. 实现系列内页：点击系列进入单本网格。
7. 保留选择模式、快捷方式入口和系列规则入口：确认长按、取消、快捷方式不回退。
8. 增加测试并运行验证。
