# EasyReader 书架小改设计

日期：2026-06-22

## 背景

本轮是在 v0.2.0 书架和系列功能基础上做小范围质量修正：

- 横向滑动切章过于灵敏，纵向滚动时容易误触。
- 单书进度条偏细，书架排序被导入/打开时间打乱。
- 归组规则入口要求用户理解 Kotlin/JVM 正则，门槛过高。
- 系列页内系统返回/边缘返回会直接退出 App，没有退回上一级。
- 导入图书、自定义规则缺少删除入口。

参考 Readest 的书库交互：排序、分组、升降序集中在视图菜单中，较少使用的功能放入折叠菜单，避免占用主工具栏。

## 设计目标

1. 降低阅读页误触切章概率。
2. 让书架排序可控、可理解，默认仍保持稳定。
3. 手动归组作为常用整理方式；自动规则保留入口，但降低使用频率和输入复杂度。
4. 系列页符合 Android 返回语义：先退回上一级，再退出 App。
5. 图书和自定义规则可以删除，并且删除前有确认。

## 非目标

- 不做完整 Readest 级别的书库系统。
- 不引入云同步、标签、搜索、发布年份等新元数据。
- 不做复杂的多层系列树；自动归大系列，手动拆小系列。

## 正则说明

当前规则中的 `(?<series>...)` 和 `(?<index>...)` 是 Kotlin/JVM 正则的命名捕获组：

- `series`：匹配出的系列名，用于自动归组。
- `index`：匹配出的排序信息，用于系列内部排序。

这不是 Python `re` 的 `(?P<series>...)` 写法。普通用户不应该必须理解这点，所以后续 UI 只把正则放到“高级规则”里。

## 书架菜单

书架首页顶部新增一个“整理”二级菜单，参考 Readest 的 `ViewMenu` 分层：

- 排序方式
  - 最近阅读，默认降序。
  - 添加日期，默认降序。
  - 标题，默认升序，自然排序。
  - 系列顺序，默认升序，优先使用系列 key 和卷号 key。
- 手动归组
  - 在选择模式下仍保留“加入系列”“移出系列”入口。
  - 在整理菜单中给出“长按选择后手动归组”的入口提示或命令。
- 自动归组规则
  - 放在整理菜单的次级入口，不作为主工具栏常驻按钮。
  - 内置规则可启停。
  - 自定义规则可新增、启停、删除。
  - 高级正则入口保留，但默认显示简单模板。

主工具栏保留“导入”。整理相关能力统一收敛到菜单，避免顶部按钮过多。

## 排序规则

新增书架排序设置，持久化到本地 DataStore：

- `Recent`：按 `lastOpenedAt ?: updatedAt` 排序。
- `Added`：按 `createdAt` 排序。
- `Title`：按标题自然排序。
- `Series`：系列条目按系列名和系列排序 key；单书按标题自然排序。

排序方向支持升序/降序。系列内部始终按卷号/自然顺序排列，不受“最近阅读”影响；这样进入系列后书籍顺序稳定。

自然排序需要识别数字段，例如：

```text
[S1_01] < [S1_02] < [S2_01] < [S5_02_01] < [S6_24.12.10]
01 < 02 < 10
```

## 归组规则

归组策略分两层：

1. 自动归大系列。
   - 适合把同一作品宇宙或同一导入批次收成一个书堆。
   - 对《魔法禁书目录》样本，`[S1_01]某魔法的禁书目录 01X`、`[S2_01]新约 某魔法的禁书目录 01X`、`[S5_02_01]某科学的超电磁炮SS 学艺都市篇X` 可由用户设置为同一个大系列“魔法禁书目录”。
   - 默认模板识别 `[S...]` 前缀，并用前缀生成排序 key。
2. 手动拆小系列。
   - 旧约、新约、创约、外传、SS 等小系列由用户手动整理。
   - 手动系列优先级最高，继续覆盖元数据和自动规则。

简单规则模板：

- 模板名：`[S编号] 前缀`
- 用户输入：大系列名。
- 匹配来源：书名。
- 归组结果：匹配 `[S...]` 的书归入用户输入的大系列。
- 排序结果：从 `[S1_01]`、`[S5_02_01]`、`[S6_24.12.10]` 提取自然排序 key。

高级规则：

- 保留当前正则能力。
- 文案说明 Kotlin/JVM 命名捕获组，避免和 Python `re` 混淆。
- 预览展示匹配出的系列名、排序 key 和命中的书籍数量。

## 阅读手势

横向切章阈值提高：

- 普通横滑距离从 `max(48dp, 屏宽 12%)` 提高到约 `max(72dp, 屏宽 18%)`。
- 快速 fling 的最小横向距离同步提高，避免短距离快速斜滑触发切章。
- 方向约束继续保留：横向位移必须显著大于纵向位移。
- 系统返回边缘区域继续保留。

## 返回行为

书架增加 BackHandler 规则：

1. 选择模式中返回：清空选择。
2. 系列页中返回：关闭系列页，回到书架首页。
3. 规则对话框或删除确认框打开时：先关闭对话框。
4. 普通书架首页返回：退出 App。

这修复“在系列页边缘返回直接退出 App”的问题。

## 删除行为

导入图书删除：

- 在选择模式中增加“删除”入口。
- 删除前弹确认框，显示选中数量。
- 删除数据库 `books` 记录；章节、阅读进度、快捷方式记录通过级联或显式清理移除。
- 删除 App 私有目录中的导入副本和封面。
- 不删除手机下载目录里的原始 EPUB。

自定义规则删除：

- 自定义规则行增加删除按钮。
- 删除前弹确认框。
- 内置规则只允许启停，不允许删除。

## 测试策略

单元测试覆盖：

- 横滑阈值提高后，旧的短横滑样例不再切章，足够长的横滑仍切章。
- `[S...]` 前缀解析自然排序 key。
- 自动大系列规则能把《魔法禁书目录》样本归成一个大系列。
- 手动系列仍覆盖自动规则。
- 不同排序方式下书架条目顺序符合预期。
- 删除自定义规则会从设置中移除。

可选设备验证：

- 真机进入系列页后，系统返回/边缘返回回到书架首页。
- 选择图书删除后，书架刷新且原始下载目录文件仍存在。
- 整理菜单可切换排序，归组规则入口仍可打开。
